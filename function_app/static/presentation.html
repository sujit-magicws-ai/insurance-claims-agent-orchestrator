Agent Processing Logic

How AI agents extract, classify, and evaluate claims

🤖 Classifier Agent

🔍 Data Extraction

1. Channel Content Parsing

  • ✓ Uses Message Extractor Module
  • ✓ Sender email/user ID
  • ✓ Issue description
  • ✓ Customer details

2. Document Intelligence

  • 📄 Calls Azure Doc Intelligence
  • ✓ Extracts PDF, Photos, Doc files
  • ✓ VIN, Odometer, Dates
  • ✓ Contract & claimant info
Merge Logic: Document values override channel values (except issue summary uses customer's own words)
🎯 Claim Classification

Contract Type Detection:

VSC Mechanical/electrical failures (engine, transmission, AC)
GAP Total loss - theft or vehicle totaled
Tire/Wheel Road hazard damage (flat, bent rim, pothole)
PPM Scheduled maintenance (oil change, filters)
Appearance Cosmetic (dents, scratches, windshield chips)
Output: Claim type, sub-type, urgency level, confidence score (0-100%)

🤖 Adjudicator Agent - VSC Evaluation Rules

📋 VSC Coverage

Covered Components:

  • ✓ Engine & Transmission
  • ✓ AC & Electrical Systems
  • ✓ Alternator & Starter
  • ✓ Roadside Assistance

Not Covered:

  • ✗ Routine Maintenance
  • ✗ Wear & Tear
  • ✗ Accident/Collision
  • ✗ Cosmetic Damage
✅ Evaluation Rules
Rule Requirement
AA-01 Contract is Active
AA-02 Within coverage period
AA-03 Mileage within limits
AA-04 Documents present
AA-05 Authorized facility
AA-06 Amount within limits
