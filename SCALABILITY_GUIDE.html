Azure Durable Functions & AI Foundry Agents

YES This Architecture is Scalable

The claims processing system built on Azure Durable Functions and Azure AI Foundry Agents is designed for enterprise-scale workloads. Both components offer automatic scaling, high availability, and can handle thousands of concurrent operations.

  • Azure Functions: Scales up to 1,000 instances automatically
  • AI Foundry Agents: Supports 100,000 messages per thread with enterprise-grade limits
  • Durable Functions: Built-in support for high-throughput orchestrations

Durable Functions Scalability

Azure AI Foundry Agents - Quotas & Limits

AI Foundry Agents have generous enterprise-grade limits:

Resource Limit Notes
Files per agent/thread 10,000 Sufficient for large document processing
Maximum file size 512 MB Per individual file
Total uploaded files size 300 GB Per agent/thread
Messages per thread 100,000 Supports long conversations
Text content per message 1,500,000 characters ~375,000 words per message
Tools per agent 128 Custom tools/functions
Vector store tokens per file 2,000,000 For RAG/file search

Rate Limits

Default Rate Limit: 10 TPS (Transactions Per Second) per resource.

To increase: Submit a support ticket via Azure Portal with business justification.

Autoscale: Can be enabled to automatically adjust rate limits based on real-time usage (disabled by default).

How to Scale

1. Azure Functions - Automatic Scaling

No action required! Azure Functions scales automatically based on:
  • Queue depth (Service Bus messages waiting)
  • HTTP request rate
  • Event backlog

2. Durable Functions - Configuration

Adjust concurrency in host.json:

{
  "extensions": {
    "durableTask": {
      "maxConcurrentActivityFunctions": 20,
      "maxConcurrentOrchestratorFunctions": 10,
      "storageProvider": {
        "partitionCount": 8
      }
    }
  }
}

3. AI Foundry Agents - Request Higher Limits

  1. Go to Azure Portal â†’ Your AI Foundry Resource
  2. Click "New Support Request"
  3. Request higher TPS with business justification
  4. Enable Autoscale feature for dynamic scaling

4. Upgrade Hosting Plan

Current Load Recommended Plan
Development / Low traffic Consumption
Production / Variable traffic Flex Consumption (Recommended)
High traffic / Low latency critical Premium (with always-ready instances)
Predictable high volume Dedicated / App Service Environment
